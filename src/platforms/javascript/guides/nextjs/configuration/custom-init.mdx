Unlike other Sentry SDKs, the Next.js SDK doesn’t have to be initialized in code, since we do it for you.
You can still customize the `Sentry.init` call, though.

## Serializable Values

Serializable values, such as strings and booleans, must be placed in the `next.config.js`:

- `module.exports.serverRuntimeConfig.sentry` for server-side code.
- `module.exports.publicRuntimeConfig.sentry` for client-side code and fallback for server-side code.

See an example setting `debug: true` in [Manual configuration](/platforms/javascript/guides/nextjs/manual-setup/manual-config#extend-default-webpack-usage-for-source-maps).

## Non-Serializable Values

Sentry’s plugin for Next.js, installed along with the SDK, contains the object where non-serializable values must be defined, such as callback functions and integrations. This must be done in the override of the `App` component:

```javascript {filename:./pages/_app.js}
import { clientConfig } from "@sentry/next-plugin-sentry";
clientConfig.beforeSend = () => {
  // ...
};
```

<Note>

Due to current Next.js limitations, you cannot set non-serializable values for only the server-side.

</Note>
